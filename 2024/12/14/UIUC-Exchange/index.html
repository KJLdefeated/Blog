這篇是交給國際處的交戰手冊。

先感謝前輩們的詳細教戰手冊，出國前我每個都讀了不下10次:

交換申請

  • 從國際處申請，在要交換的那個學期的前一年開始申請 ex.112-1申請113-1的交換
  • 主要看在校成績，語言成績達標就好(TOEFL 100)
  • TOEFL超過100或接近100應該都可以（我只有考99）
  • 100不好考，可以提早準備
  • 國際處和電機資訊學院都有獎學金可以申請
    交大這邊結果大概學期末會公布，UIUC會在三月開始請你填一些申請資料和準備要修的課之類的。

流程

  • 填寫簽證申請表DS-160
  • 申請SEVIS I-901並上網繳費（可以刷卡）
  • 預約面談時間、申請J1(六月七月是申請季，可以盡量提早搞定)
  • 繳交J1簽證費及資料處理費
  • 備好文件面試，包含
    • DS-160確認單
    • 預約確認單
    • 繳費收據
    • 有效護照
    • DS-2019正本
    • SEVIS繳費證明
    • 所有的舊護照（如果遺失了要去移民署申請出入境證明）
    • 美簽規格的大頭照
    • 若改過名要帶戶籍謄本（我改過名但我忘記帶，他還是讓我過了）
    • 在校成績單和托福成績單

上面的東西我好像只用到護照和DS-160，建議可以提早到美國在台協會，不然會很多人要排隊，手機會被沒收一直站著發呆很無聊QQ，
我當時還遲到排在最後一個，很多人都西裝鼻挺，但我只有穿短褲短袖，我還蠻緊張會不會被趕出去，但其實沒差，面試官只問了一下去那邊讀什麼，誰資助你之類的，最後他會把你的護照收走，過幾天就可以拿到有簽證的護照了。

行

UIUC的校園很大，教室和居住的地方可能走路需要十幾二十分鐘，搞個腳踏車或電動滑板車應該蠻方便的，不過冬天下雪可能就不太方便。學校裡有公車，學生可以免費搭，不過路線對我們來說不太方便。聽說二手車只需要台幣不到十萬就可以整到一台，有車就是自由，也可以考慮一下，尤其玉米田特別無聊，開車出去晃晃應該蠻棒的。

樂

我沒有去Club或Bar之類的，不過倒是有去過社團，可以提供點意見。UIUC的社團非常非常多，大學部學生也很熱衷社團，在這裏讀大學應該非常好玩，因為有很多社團可以參加，大家也很熱情。我有去Electronic Music Club，也就是電音社，參加過幾次社課，大家都很友善，而且美國當地人居多，跟在電資學院修課的人種組成（東亞印度人）不一樣，社團裡比較多金髮碧眼、黑人，在這裏體驗當地社交、英文口說的機會比較多，很推薦大家去找幾個來參加。

修課

選課，UIUC用課號代表課程，如CS123, CS408…，課號開頭分為1, 2, 3, 4, 5，通常1,2給大一二修，3,4給大二三修，4和5給研究所的修，大四呢？唉本科大四早就去實習了誰跟你修課（誤），不過我這學期體感大四生真的比較少（除了交換生），對於交換生而言，因為沒有prerequisite，所以選課上可能會被擋修，不過開學前UIUC的國際處會幫大家選到自己想修的課，不過5開頭就比較麻煩，因為需要老師和系所確認，被一直寄信問說確定要不要修，害我以為loading會很大之類的，但其實修下來還好。這學期修的分別是CS341、CS483、CS542。標題連結是Course work的Github。

CS342 System Programming

主要是教OS的概念和實作，內容有C語言、Memory Allocation、Processes、Threads、Virtual Memory、File System和Networking，是很扎實的System入門課，對沒有相關經驗的人應該很硬，因為這堂課每週有兩次作業，一次Lab一次Machine Problem，都是要實作OS裡面的某個概念(ex. malloc, TLB, file system)，作業設計得很好，助教只讓學生實作核心演算法的部分，其他部分是打包好的，除了一些MP要從頭自己刻以外，其他都有包好只需要做TODO。老師上課很生動有趣，不像以往的課程老師都是站在講台邊講解，這門課的老師上課就像一場表演，他會很熱情的跟你介紹C語言是什麼，指標是什麼，好像非常神奇非常有趣，老師上課時也會活蹦亂跳跑來跑去點人問問題，這大概是我很culture shock的一個地方，因為我第一次見到這種上課模式。對於大學部的學生而言，這是蠻有名也蠻硬的一堂課，因為我跟其他人聊天時，聊到我有修341，他們都會wow你修了341，那是不是很難？但我整學期修下來感覺不會很難，類比交大的網路程式設計那種感覺吧，就是很實作的課，但感覺也沒有NASA和高等Unix那麼難。總之是堂很好的課，讓我複習了OS的概念和實作，這堂課的Course Website也很好，提供了很多資源。順帶一提，我也參加了這堂課的Honor Section，他會講一些更進階的東西（Distributed System, Kernel Programming, Embedded System），期末要做出一個有關Embedded System的Projet，我們這組搞了一個File system，去抓Course Explorer的資料，我跟一個印度大二和台裔美國大二組隊，他們感覺都好電，我負責躺平。

CS483 Applied Parallel Programming

這堂課主要專注在NVIDIA CUDA上的應用優化，各種CUDA的技巧(Matrix Multiplication, Convolution的Optimization)，其他Parallel Programming (like POSIX threads)的東西不會碰，修課的台灣人很多，這堂課我看以前很多來交換的人都會修，讀MCS的人也會修，大家可以一起抱團取暖，但我個人覺得這堂課沒有到那麼頂，以老師上課方式來說，上課方式跟台灣比較像，就是依照投影片講解，很少課堂上的Demo和與學生互動，作業方面，總共有八次lab，我個人覺得比較無聊一點，參考一下講義就可以寫，而且跟前幾年差不多，沒什麼變化的感覺，但我還是從這堂課上學到很多，CUDA現在是顯學，大家都在用，學一下不會虧，像我在找下學期實習的面試就剛好用到了。修了這堂課讓我更了解GPU的work，也了解Deep Learning System想考量的地方，比方說kernel和memory之間data transfer的overheads，感覺是個可以繼續學習的領域。

CS542 Statistical Reinforcement Learning

神課，我就是為了這堂課來交換的（誤）。Reinforcement Learning Theory的課，雖然開在CS，但根本是數學課，老師第一堂課屆就說了「Zero implementation. Only mathematical proof.」，這堂課詳細介紹RL理論的入門，從Offline RL到Online RL的分析，讓我們從數學的角度去看RL，這堂課有點像Theory Computer Science，就是計算理論，我們會分析演算法的Sample Complexity，也就是要採樣幾次演算法才會收斂？我們的data需要什麼假設，Learning才有辦法converge？什麼Case可以讓當今最強的RL演算法都沒辦法work？我之前是做Empirical RL相關的研究，修完這堂課之後知道理論RL在做什麼，考量的點跟Empirical有多麽不一樣，也知道我大概不喜歡做理論XD。雖然如此，但上課還是非常有趣，老師是中國人，講解非常有邏輯，一個自己讀起來非常難的概念，他就是能講解得非常直覺非常trivial，因為是數學課，老師上課會直接demo理論證明分析，一步一步地推導都很trivial，讓人全程都能跟上。修這堂課之前我的數學基礎算是很薄弱的，我只修過微積分、線代、機率和Empirical的RL，這堂課雖然包含一點Statics，但我覺得只需要機率和線代就非常夠用！Course works包含四次作業，只需要線代和機率就寫得出來了，Final Project是要reproduce一篇paper，我是選Offline RL Theory，也讓我知道我可能不太適合寫paper XD。總而言之，雖然我在這堂課學到的不算是什麼通用的skill，也沒有做出特別厲害的project，但還是很開心能夠學習到這些。

總結下來，我覺得UIUC的課程都很棒，加上Faculty數量比交大多很多，有很多領域都有開課，像Theory方面的課就比交大多很多，不愧是全美Top5的CS School，課程品質也很不錯，很注重實作，也很就業導向，在大學部修課很多人都在討論就業面試相關的話題，不過我覺得跟交大比較的話，交大還是不會輸的，交大的課程水準也很不錯的。
